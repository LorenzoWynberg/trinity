# Learnings

Consolidated patterns, gotchas, and best practices discovered during development.

## Structure

Organize learnings by topic:

- `go.md` - Go language patterns and idioms
- `cobra.md` - CLI framework patterns
- `sqlite.md` - Database patterns
- `testing.md` - Testing strategies
- `git.md` - Git workflow learnings

## Format

Each file should contain:

```markdown
# Topic Name

## Patterns

### Pattern Name
Brief description and example.

## Gotchas

### Issue Description
What went wrong and how to avoid it.

## Best Practices

- Practice 1
- Practice 2
```

## Maintenance

1. Extract learnings from activity logs before archiving
2. Update existing files when discovering corrections
3. Remove outdated information
4. Keep entries concise and actionable
